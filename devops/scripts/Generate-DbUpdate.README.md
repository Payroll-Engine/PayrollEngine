# Generate-DbUpdate

Generates a complete, self-contained `Update-Model.sql` from `DbVersion.json` in one step:

1. Reads `DbVersion.json` for `OldVersion`, `NewVersion` and `UpdateDescription`
2. Formats `History\v<OldVersion>\Create-Model.sql`  (baseline)
3. Formats `Create-Model.sql`  (current)
4. Diffs both via `Compare-DbScript.ps1`  →  raw delta SQL
5. Wraps delta with VERSION_CHECK header and VERSION_SET footer
6. Writes `Update-Model.sql`

## Output Structure

```sql
-- VERSION_CHECK: validates DB is on OldVersion, aborts via SET NOEXEC ON if not

USE [PayrollEngine]; GO
SET XACT_ABORT ON; GO
...version check...
SET NOEXEC ON  -- on mismatch

-- DB_SCRIPTS: delta DDL inside a transaction

BEGIN TRANSACTION; GO
USE [PayrollEngine]; GO
...DROP / CREATE / TODO comments...

-- VERSION_SET: inserts NewVersion record and commits

INSERT INTO dbo.[Version] ...; GO
COMMIT TRANSACTION; GO
SET NOEXEC OFF; GO
```

Delta content:
- `DROP` + `CREATE` for modified stored procedures, functions, views
- `CREATE` for new objects
- `DROP` for removed objects
- `-- TODO` comments for modified tables (manual `ALTER TABLE` required)

> **Note:** SQL Server DDL is transactional — `CREATE`/`DROP`/`ALTER` are rolled back
> automatically if `XACT_ABORT ON` fires on any error.

## Config File

**`DbVersion.json`** — placed in the `Database\` folder, committed with the SQL scripts:

```json
{
  "OldVersion":          "0.9.5",
  "NewVersion":          "0.9.6",
  "CreateDescription":   "Payroll Engine: Full setup v0.9.6",
  "UpdateDescription":   "Payroll Engine: Migration v0.9.5 -> v0.9.6"
}
```

| Field | Purpose |
|-------|---------|
| `OldVersion` | Expected version in the target DB — used for VERSION_CHECK and to locate `History\v<OldVersion>\` |
| `NewVersion` | Version written after successful migration — used in VERSION_SET |
| `CreateDescription` | Description for the version row in `Create-Model.sql` (managed manually) |
| `UpdateDescription` | Description for the version row in `Update-Model.sql` |

## Directory Layout

```
PayrollEngine.Backend\Database\
  DbVersion.json               ← version config (read by this script)
  Create-Model.sql             ← current schema (diff target)
  Update-Model.sql             ← generated by this script
  Drop-Model.sql               ← not touched
  History\
    v0.9.5\
      Create-Model.sql         ← diff baseline (read-only)
      Update-Model.sql
      Drop-Model.sql
      DbVersion.json
```

## Parameters

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `-DatabaseDir` | No | Caller's working directory | Directory containing the SQL files and `History\` |
| `-ConfigFile` | No | `DbVersion.json` in DatabaseDir | Path to the JSON config file |
| `-ScriptsDir` | No | Script's own directory | Directory containing `Format-DbScript.ps1` and `Compare-DbScript.ps1` |
| `-BaselineFile` | No | `History\v<OldVersion>\Create-Model.sql` | Override the baseline SQL file |
| `-CurrentFile` | No | `Create-Model.sql` in DatabaseDir | Override the current SQL file |
| `-TargetFile` | No | `Update-Model.sql` in DatabaseDir | Override the output path |
| `-KeepTemp` | No | `false` | Keep intermediate formatted files |
| `-DryRun` | No | `false` | Validate inputs and print plan without writing |

## Usage Examples

```powershell
# Run from the Database directory (all defaults)
cd Backend\Database
..\..\devops\scripts\Generate-DbUpdate.ps1

# Explicit paths
Generate-DbUpdate.ps1 `
    -DatabaseDir C:\...\Database `
    -ScriptsDir  C:\...\devops\scripts

# Preview without writing anything
Generate-DbUpdate.ps1 -DryRun

# Keep intermediate formatted files for inspection
Generate-DbUpdate.ps1 -KeepTemp
```

## Typical Release Workflow

```powershell
cd Backend\Database

# 1. Generate Update-Model.sql
..\..\devops\scripts\Generate-DbUpdate.ps1

# 2. Review Update-Model.sql
#    Replace any '-- TODO' table comments with correct ALTER TABLE statements
#    Re-run if the schema diff needs refreshing

# 3. After release: snapshot current state into History
mkdir History\v0.9.6
copy Create-Model.sql  History\v0.9.6\Create-Model.sql
copy Update-Model.sql  History\v0.9.6\Update-Model.sql
copy Drop-Model.sql    History\v0.9.6\Drop-Model.sql
copy DbVersion.json    History\v0.9.6\DbVersion.json

# 4. Advance DbVersion.json for the next release cycle
#    OldVersion = "0.9.6", NewVersion = "0.9.7"
```

## Dependencies

`Generate-DbUpdate.ps1` calls `Format-DbScript.ps1` and `Compare-DbScript.ps1`.
All three scripts must be in the same directory (or override with `-ScriptsDir`).

## Temp Files

Intermediate formatted files are written to a timestamped sub-folder of `$env:TEMP`
and deleted automatically. Use `-KeepTemp` to retain them for inspection.

Formatted and temp files are not committed — add to `.gitignore`:

```
*.Formatted.sql
Delta.Raw.sql
```
